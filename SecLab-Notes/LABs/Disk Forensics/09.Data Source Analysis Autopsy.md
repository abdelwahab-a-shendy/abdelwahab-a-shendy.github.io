
# 🧪 **Analysis of Data Source Using Autopsy – Autopsy Forensic Tool**

***

## 🎯 **Main Idea**

Once a disk image is loaded, **Autopsy** automatically begins analyzing it using a series of **modules**, and the results are displayed in a clean graphical interface.

<Image src="https://cdn.hashnode.com/res/hashnode/image/upload/v1760448763208/5fd9b9d9-d41c-4d66-9406-db9596dd49ce.png" align="center" fullwidth="false" />

***

## 🔄 **Key Terminology in Autopsy**

| Term               | Description                                                            |
| ------------------ | ---------------------------------------------------------------------- |
| **Data Source**    | The original digital image being analyzed (e.g., disk clone or image). |
| **File Views**     | Different ways to explore files inside the image.                      |
| **Ingest Results** | Results generated by automatic analysis when the data source is added. |
| **Tags**           | Labels added by the investigator to mark important files or content.   |
| **Timeline**       | A chronological view of system events.                                 |

***

## 🧱 **Analysis Structure in Autopsy**

### 1. **Importing Data Source (Original Image)**

* You import a disk image (`.E01` or `.img`) as the starting point.

* You can browse the content as if navigating a live system.

* Filtering options include:

  * **Extension**

  * **File Header**

  * **File Size**

***

### 2. **Ingest Results**

Autopsy automatically analyzes the image and extracts:

#### 📂 **Recovered Files (File Carving)**

* Extracts files even if deleted or corrupted.

* Includes images, documents, system files.

* Some files may appear with size "0" → indicates failed recovery.

###### Example:

Autopsy recovered 3 images using File Carving and showed them under the *Images* tab.

###### ♻️ **Deleted / Recycle Bin Items**

* Identifies files deleted from Recycle Bin.

* Shows deletion/modification timestamps.

#### 🛑 **How to Identify Deleted Files?**

* Deleted files often begin with an underscore `_`.

* This is due to how file systems (e.g., NTFS) mark deleted files.

🧠 **DF Tip**:

> When a file is deleted in Windows, its first character is replaced with `_`, but its data remains until overwritten.

***

## 📧 **Email and Keyword Extraction**

Autopsy can extract:

* Email addresses from scripts, configs, or logs (e.g., inside netcat or SQL scripts).

* Keywords that appear in context.

Example:

> Two email addresses were extracted from a netcat-related SQL script, appearing in a help message from the developer.

***

## 🖼️ **Image and Video Analysis**

### 1. **Image/Video Analysis**

* Found under a dedicated tab.

* You can:

  * Browse recovered media.

  * Identify if it's user-generated or system-generated.

### 2. **Geo-location Metadata**

* If images include GPS metadata, Autopsy displays the geographic data.

***

## 🗓️ **Timeline Analysis**

One of the most powerful features in Autopsy.

### ✅ Why Is It Important?

Understanding the **sequence of events** helps determine:

* When a website was visited.

* When a tool was executed.

* When a file was deleted.

### 🧠 **Timeline Analysis Examples**

| Date | Event                                                   |
| ---- | ------------------------------------------------------- |
| 2011 | File named "cat" was created — later marked as deleted. |
| 2023 | Tools were modified; Recycle Bin accessed.              |

🔍 All extracted events are shown as **Time Points**.\
📌 Always **set the correct time zone** for accurate results.

***

## 🔁 **Discovery Tool – Advanced Search**

* To re-analyze a file group (e.g., images, videos, logs), use the **Discovery** tab to filter and review results.

***

## 📊 **Generating Final Report**

*After analysis is complete:*

### 📋 **Steps to Generate a Report**

1. Go to `Generate Report`.

2. Choose:

   * Report type: HTML, Excel.

   * Report name (e.g., "Draft\_Analysis").

   * What to include:

     * Tags only,

     * Custom results,

     * or All analysis results.

3. Report is saved under `/Reports` inside the case folder.

🔎 The report includes:

* Original file paths

* Extracted keywords

* Images and media

* Tags

* Timestamps

***

## 🧠 **Pro Tips for Using Autopsy**

| 💡  | Tip                                                         |
| --- | ----------------------------------------------------------- |
| ⏱️  | Use the Timeline to understand event sequences.             |
| 🔍  | Always start with Ingest Results for a high-level overview. |
| 🧹  | Filter files by type or size to save time.                  |
| 🗑️ | Review zero-byte files — they may be corrupted evidence.    |
| 🏷️ | Use Tags and notes to organize your findings.               |

# 🧪 **تحليل مصدر البيانات باستخدام Autopsy – Analysis of Data Source**

*Analysis of Data Source in Autopsy Forensic Tool*

***

## 🎯 **الفكرة الأساسية**

بمجرد إدخال صورة قرص صلب (Disk Image)، تبدأ أداة **Autopsy** بتحليلها تلقائيًا باستخدام مجموعة من الموديولات (Modules)، وتُعرض النتائج في واجهة رسومية منظمة.

***

## 🔄 **مصطلحات أساسية داخل Autopsy**

| المصطلح            | الوصف                                                      |
| ------------------ | ---------------------------------------------------------- |
| **Data Source**    | الصورة الرقمية الأصلية التي تم تحليلها (مثل: نسخة من قرص). |
| **File Views**     | طرق مختلفة لاستعراض الملفات داخل الصورة.                   |
| **Ingest Results** | نتائج التحليل التلقائي الذي يتم بمجرد إدخال المصدر.        |
| **Tags**           | وسوم يضعها المحقق على ملفات أو كلمات مهمة.                 |
| **Timeline**       | عرض زمني لتسلسل الأحداث داخل الجهاز.                       |

***

## 🧱 **هيكل التحليل داخل Autopsy**

### 1. **الـ إدخال مصدر البيانات Data Source (الصورة الأصلية)**

* يتم استيراد صورة القرص (`.E01` أو `.img`) كنقطة البداية , تحتوي على ملفات القرص الأصلية.

* يمكنك تصفحها بحرية كما لو كنت تتصفح الجهاز الحقيقي.

* يدعم الفلترة حسب :

  * **الامتداد (Extension)**

  * **رأس الملف (File Header)**

  * **الحجم (Size)**

***

### 2. **نتائج التحليل (Ingest Results)**

Autopsy تلقائيًا يحلل الصورة ويستخرج:

#### 📂 **Recovered Files (File Carving) (استخراج الملفات المحذوفة)**

* يستخرج الملفات حتى لو كانت محذوفة أو تالفة.

* مثل الصور، المستندات، ملفات النظام.

* بعض الملفات تظهر بحجم "0" → هذا يعني أنه لم يستطع استعادتها بشكل كامل (فشل الاستعادة)

###### مثال :

استخرج Autopsy 3 صور عبر Carving وظهرت في تبويب الصور.

> تم استخراج 3 صور باستخدام الـ File Carving\
> وظهرت أيضًا في تبويب الصور (Images)

###### ال♻️ **Deleted / Recycle Bin Items**

* يتم تحديد الملفات التي تم حذفها من Recycle Bin.

* يوضح وقت الحذف أو التعديل.

#### 🛑 **كيف تتعرف على الملفات المحذوفة؟**

* الملفات المحذوفة تظهر غالبًا بـ **أسماء تبدأ بـ Underscore (\_)**

* هذا جزء من طريقة نظام الملفات (مثل NTFS) لتعليم الملف كمحذوف.

🧠 **معلومة DF مهمة**:

> عندما يُحذف ملف في Windows، يتم إزالة أول حرف من اسمه واستبداله بـ `_filename.ext`، لكن تظل بياناته موجودة حتى يتم استبدالها.

***

## 📧 **البحث عن البريد الإلكتروني والـ Keywords**

Autopsy يستطيع استخراج:

* عناوين بريد إلكتروني من الملفات أو الأكواد (مثلاً داخل ملف netcat أو سكربت).

* كلمات مفتاحية (Keywords) تظهر من السياق.

مثال:

> تم استخراج عنواني بريد إلكتروني من ملف خاص بـ netcat داخل سكربت SQL، وقد ظهر ضمن help message للمطور.

***

## 🖼️ **تحليل الصور والفيديوهات**

### 1. **Image/Video Analysis (الصور والفيديوهات)**:

* يظهر في تبويب خاص.

* يمكن من خلاله:

  * تصفح الصور المستخرجة.

  * معرفة ما إذا كانت الصور حقيقية أو System Generated.

### 2. **Geo-location (الموقع الجغرافي)**:

* لو الصور تحتوي على بيانات موقع (GPS Metadata)، Autopsy يعرضها.

***

## 🗓️ **تحليل الـ Timeline (الخط الزمني)**

واحدة من أقوى الأدوات داخل Autopsy.

### ✅ لماذا هو مهم؟

لأن فهم **تسلسل الأحداث** يساعدك على تحديد:

* وقت الدخول على موقع.

* وقت تشغيل أداة.

* وقت حذف ملف.

### 🧠 **أمثلة على Timeline Analysis**:

| التاريخ | الحدث                                               |
| ------- | --------------------------------------------------- |
| 2011    | تم إنشاء ملف باسم "cat" – تبين لاحقًا أنه ملف محذوف |
| 2023    | تم تعديل أدوات، وتم الوصول إلى Recycle Bin          |

🔍 أي حدث يتم استخراجه يظهر في شكل "Time Point"\
📌 **حدد المنطقة الزمنية (Time Zone)** بدقة للحصول على Timeline دقيق.

***

## 🔁 **Discovery Tool – البحث داخل النتائج**

* لو أردت إعادة تحليل مجموعة ملفات (صور، فيديو، لوجات):

* استخدم تبويب **Discovery** لتصفية النتائج ومراجعتها.

***

## 📊 **توليد التقرير النهائي (Generate Report)**

*بعد انتهاء التحليل* :

### 📋 **خطوات إنشاء التقرير:**

1. اذهب إلى `Generate Report`

2. اختر:

   * نوع التقرير: HTML, Excel

   * اسم التقرير (مثلاً: "Draft\_Analysis")

   * ما تريد تضمينه:

     * ال Tags فقط

     * نتائج مخصصة

     * أو كل نتائج التحليل

3. يتم حفظه تلقائيًا في مجلد `/Reports` داخل القضية.

🔎 التقرير يحتوي على:

* المسارات الأصلية للملفات

* الكلمات المفتاحية المستخرجة

* الصور والفيديوهات

* الوسوم (Tags)

* التواريخ

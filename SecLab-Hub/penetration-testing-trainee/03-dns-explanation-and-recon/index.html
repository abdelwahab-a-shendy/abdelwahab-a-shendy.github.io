    03 dns explanation and recon

    🧩 Introduction

    As mentioned earlier, the purpose of the DNS (Domain Name System) is to make accessing websites easier.\ Instead of memorizing each website’s IP address, we simply use a domain name like google.com.

    But the question is 👇\ How does this system actually work?

    🔍 First: Components of the DNS System

    Before understanding the process, we need to get familiar with a few key terms:

    🧠 1. Recursive Resolver (or Recursive DNS Server)

    • This is the server that belongs to your Internet Service Provider (ISP).

    • Its role is to search for the IP address of the domain you type into your browser.

    • In other words, it starts the lookup process step by step on your behalf.

    🌐 2. Root DNS Nameserver

    • There are 13 main Root Servers around the world.

    • They are considered the backbone of the DNS service across the Internet.

    • Each one contains information about all Top-Level Domains (TLDs) such as:

    • .com

    • .net

    • .org

    • and others.

    🏷️ 3. TLD (Top-Level Domain DNS Nameserver)

    • These servers contain the addresses of the servers responsible for specific domains.

    • They point to the server that actually holds the final IP address.

    • These are usually called Authoritative Nameservers.

    🖥️ 4. Authoritative Nameserver

    • This is the server responsible for storing the real IP address of the domain.

    • When the request reaches it, it responds with the final IP address of the website.

    📊 Illustration of the DNS Query Process :

    User → Recursive Resolver → Root Server → TLD Server → Authoritative Server → IP Address

    📘 Second: DNS Records

    Each domain contains a set of records, each with a specific purpose:

    Record Type Description
    A Record Contains the website’s IPv4 address.
    AAAA Record Contains the website’s IPv6 address.
    CNAME Record Used when a domain is an alias for another domain.
    MX Record Contains the Mail Server address used for email.
    TXT Record Contains notes or settings such as SPF Records for email authentication.
    SOA Record Contains information such as the administrator’s email address.
    NS Record Points to the domain’s Authoritative Nameserver.

    🧠 Third: Domain Information Gathering

    🔹 1. Get basic information about a domain:

    nslookup yahoo.com

    🔹 2. Get details for a specific record (e.g., MX):

    nslookup -type=MX yahoo.com

    🔹 3. Using the dig tool (more detailed):

    📌 To retrieve all available information:

    dig yahoo.com any

    📌 To retrieve a specific record type (e.g., MX only):

    dig yahoo.com MX

    📌 To display the full query trace:

    dig +trace yahoo.com

    🌐 4. Useful Tools

    🔸 DigWebInterface : https://digwebinterface.com/

    An online website that allows you to perform the same dig commands easily and retrieve information about any domain.

    🧾 Fourth: WHOIS Information

    The WHOIS service provides detailed information about a domain, including:

    🔸 whoIs: https://who.is/

    • Owner’s name

    • Email address

    • Creation and expiration dates

    • Registrar company (the organization responsible for registering the domain)

    🔹 Example:

    whois yahoo.com

    🧩 المقدمة

    كما ذكرنا سابقًا، الهدف من نظام DNS (Domain Name System) هو تسهيل الوصول إلى المواقع الإلكترونية.\ بدلًا من حفظ عنوان الـ IP لكل موقع، نستخدم اسم نطاق بسيط مثل google.com.

    لكن السؤال هو 👇\ كيف يعمل هذا النظام فعليًا؟

    🔍 أولًا: مكونات نظام الـ DNS

    قبل أن نفهم عملية عمله، يجب أن نتعرف على بعض المصطلحات الأساسية:

    🧠 1. الـ Recursive Resolver (أو Recursive DNS Server)

    هو الخادم التابع لمزود خدمة الإنترنت (ISP) الخاص بك.\ وظيفته هي البحث عن عنوان IP للنطاق الذي تكتبه في المتصفح.

    بمعنى آخر، هو الذي يبدأ عملية البحث خطوة بخطوة نيابةً عنك.

    🌐 2. خادم الجذر (Root DNS Nameserver)

    يوجد حول العالم 13 خادم جذر رئيسي.\ تُعتبر هذه الخوادم العمود الفقري لخدمة DNS على الإنترنت.

    كل خادم منها يحتوي على معلومات عن جميع النطاقات العليا (Top-Level Domains - TLDs) مثل:

    • .com

    • .net

    • .org\ وغيرها.

    🏷️ 3. خوادم النطاق الأعلى (TLD DNS Nameserver)

    تحتوي هذه الخوادم على عناوين الخوادم المسؤولة عن نطاقات معينة.\ وهي التي تُشير إلى الخادم الذي يحتوي فعليًا على عنوان الـ IP النهائي.

    عادةً ما تُسمى هذه الخوادم بـ Authoritative Nameservers.

    🖥️ 4. الخادم الموثوق (Authoritative Nameserver)

    هو الخادم المسؤول عن تخزين عنوان IP الحقيقي للموقع.\ عندما تصل إليه الاستعلامات، فإنه يرد بعنوان الـ IP النهائي للموقع.

    📊 رسم توضيحي لعملية استعلام DNS:

    المستخدم → الـ Recursive Resolver → خادم الجذر → خادم TLD → الخادم الموثوق → عنوان الـ IP

    📘 ثانيًا: سجلات الـ DNS (DNS Records)

    كل نطاق يحتوي على مجموعة من السجلات، ولكل سجل وظيفة محددة:

    نوع السجل الوصف
    A Record يحتوي على عنوان الـ IPv4 الخاص بالموقع.
    AAAA Record يحتوي على عنوان الـ IPv6 الخاص بالموقع.
    CNAME Record يُستخدم عندما يكون النطاق اسمًا مستعارًا لنطاق آخر.
    MX Record يحتوي على عنوان خادم البريد الإلكتروني للموقع.
    TXT Record يحتوي على ملاحظات أو إعدادات مثل سجلات SPF للتحقق من البريد الإلكتروني.
    SOA Record يحتوي على معلومات مثل بريد مدير النظام.
    NS Record يُشير إلى الخادم الموثوق (Authoritative Nameserver) الخاص بالنطاق.

    🧠 ثالثًا: جمع معلومات عن النطاق (Domain Information Gathering)

    🔹 1. للحصول على معلومات أساسية عن النطاق:

    nslookup yahoo.com

    🔹 2. للحصول على تفاصيل سجل محدد (مثل سجل البريد MX):

    nslookup -type=MX yahoo.com

    🔹 3. باستخدام أداة dig (معلومات أكثر تفصيلًا):

    📌 لجلب جميع المعلومات المتاحة:

    dig yahoo.com any

    📌 لجلب نوع محدد من السجلات (مثل MX فقط):

    dig yahoo.com MX

    📌 لعرض التتبع الكامل للاستعلام:

    dig +trace yahoo.com

    🌐 4. أدوات مفيدة على الإنترنت

    🔸 DigWebInterface: https://digwebinterface.com/\ موقع إلكتروني يتيح لك تنفيذ أوامر dig بسهولة،\ واسترجاع معلومات عن أي نطاق دون الحاجة لأوامر طرفية.

    🧾 رابعًا: معلومات WHOIS

    خدمة WHOIS تُتيح لك معرفة معلومات تفصيلية عن أي نطاق، مثل:

    • اسم مالك النطاق

    • عنوان البريد الإلكتروني

    • تاريخ الإنشاء والانتهاء

    • شركة التسجيل (Registrar) المسؤولة عن النطاق

    🔹 مثال عملي:

    whois yahoo.com

    🔸 موقع للاستعلام عن WHOIS: https://who.is/
